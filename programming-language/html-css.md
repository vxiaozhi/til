# CSS

## Tailwind CSS、Bootstrap、shadcn/ui 详细对比

| 维度 | **Tailwind CSS** | **Bootstrap** | **shadcn/ui** |
|------|-------------------|---------------|---------------|
| **类型** | 实用优先的原子化 CSS 框架 | 完整的组件库框架 | 基于组件的代码库（非npm包） |
| **核心理念** | 实用优先，低级别CSS工具类 | 预制的响应式组件和工具 | 复制粘贴高质量组件代码到项目中 |
| **安装方式** | npm 安装，需要配置构建工具 | CDN引入或npm安装 | 无安装，通过命令行工具复制组件代码 |
| **自定义性** | 极高，可配置设计系统 | 中等，通过变量覆盖 | 极高，直接编辑复制的组件代码 |
| **学习曲线** | 中等，需记忆工具类 | 较低，组件API明确 | 中等，需理解React组件结构 |
| **样式控制** | 内联工具类组合 | 预定义类名+组件结构 | 完整的组件源代码控制 |
| **捆绑大小** | 很小（通过摇树优化） | 中等（可模块化导入） | 零依赖，仅包含使用的组件代码 |
| **JavaScript依赖** | 无 | 依赖jQuery（v4）或原生JS（v5） | 基于React，需React环境 |
| **主题定制** | 通过tailwind.config.js配置 | 通过SASS变量覆盖 | 通过CSS变量或直接修改代码 |
| **设计系统** | 可完全自定义 | 预设的Bootstrap设计语言 | 基于Tailwind，可扩展设计 |
| **组件完整性** | 无内置组件，需自行构建 | 完整组件生态系统 | 核心组件库，持续扩展中 |
| **社区生态** | 极大，插件和模板丰富 | 极大，历史最悠久的框架 | 快速增长，专注于质量 |
| **响应式设计** | 移动优先，内建断点前缀 | 移动优先，响应式网格系统 | 基于Tailwind的响应式工具 |
| **性能影响** | 极低，生成最小CSS | 中等，包含未使用的样式 | 最佳，仅包含使用的代码 |
| **使用场景** | 高度定制化的现代Web应用 | 快速原型、企业级应用 | 需要高质量UI的React应用 |
| **维护方式** | 版本升级，工具类变更 | 版本升级，API可能变更 | 手动更新复制的组件代码 |

**关键区别总结：**
- **Tailwind CSS**：提供构建UI的工具，而非现成组件
- **Bootstrap**：提供完整的UI解决方案，开箱即用
- **shadcn/ui**：提供可完全控制的组件代码，无运行时依赖
